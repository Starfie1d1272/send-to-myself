# SendToMyself — 需求与边界（SPEC V1）

> 本文件是项目的**单一事实来源（single source of truth）**。
> 任何关于「做不做、怎么做」的争议，以本文件为准。新增范围必须先改本文件。

最后更新：2026-06-19

---

## 0. 一句话定位

部署在个人 NAS、单用户、跨平台的「发给自己」**统一信息流**，替代「微信发给自己」，但搜索和后续处理更强。

**它不是**：笔记软件、密码管理器、完整任务管理器、网盘。
**它是**：一个统一的「发给自己」入口。

---

## 1. 三条核心设计原则

1. **发送时不分类** — 输入/粘贴 → 一次点击发送。发送前不强制选文件夹/类型/标签/是否临时/是否敏感/是否待办。
2. **查看时再分类** — 所有内容进同一条倒序时间线，事后再标记（待办 / DDL / 想法 / 稍后看 / 置顶 / 归档删除）。
3. **一种数据，多种视图** — 底层只有 `Item`(+`Attachment`)，不拆成笔记/文件/任务/剪贴板/密钥独立模块。页面按属性筛选。

---

## 2. 内容形态

系统**自动识别**展示类型，用户无需选择：`text` / `link` / `image` / `file`。

一条记录可以**混合**：文字 + 一个或多个链接 + 一个或多个附件。

> 示例：`地球化学作业，完成 11.5.2，6 月 25 日前提交` + 附件 `作业要求.pdf`

---

## 3. 数据模型（契约草案，最终以 packages/shared 为准）

```ts
type Item = {
  id: string;
  content: string;

  kind: "text" | "link" | "image" | "file";
  category: "none" | "todo" | "idea" | "read_later"; // 主标记，单选

  completed: boolean;
  pinned: boolean;

  dueAt?: string;       // DDL
  completedAt?: string;

  createdAt: string;
  updatedAt: string;
};

type Attachment = {
  id: string;
  itemId: string;
  filename: string;     // 保留原始文件名与扩展名
  mimeType: string;     // 保留 MIME
  size: number;
  path: string;         // NAS 挂载目录中的相对路径，不入库存二进制
  createdAt: string;
};
```

- 链接预览信息：V1 先直接放在 `Item` 上（或单独 `LinkMetadata` 表，实现时再定）。
- 附件**只存路径**，二进制存 NAS volume，不进数据库。

---

## 4. 分类与状态（极简）

- **主标记（单选，最多一个）**：`none` / `todo` / `idea` / `read_later`
- **独立状态**：`pinned`、`completed`
- **DDL 分桶（仅展示，不做完整日历）**：已过期 / 今天截止 / 明天截止 / 本周截止 / 更晚截止

**V1 不做**：多级文件夹、复杂标签树、多层项目、双向链接、知识图谱。自由标签留作后续增强。

---

## 5. 待办与 DDL

待办是普通记录的**附加属性**，不建独立任务系统。

- 字段：`category=todo`、`completed`、`dueAt`、`completedAt`
- 操作：一键转待办；点两下设 DDL；快捷日期（今天/明天/本周日/自定义）；勾选完成；改/清 DDL
- **不做**：子任务、重复任务、任务依赖、甘特图、看板、课程表、月历视图、多人指派

---

## 6. 自动识别（V1 纯规则，无 AI）

仅用于：**展示优化 / 提供快捷操作 / 提示可能的分类**。不得强改内容或自动删信息。

| 识别 | 规则 | 作用 |
|---|---|---|
| 链接 | URL 正则 | 抓取标题/域名/favicon/描述/封面 |
| 图片/文件 | 上传内容 | 缩略图 / 文件信息 |
| 疑似待办 | `TODO` / `DDL` / `截止` 等关键词 | 提示转待办 |
| 疑似 DDL | 日期解析（`6月25日前`/`本周五`/`明晚23:59`/`DDL 6.30`/`截止：2026-06-30`） | **仅建议，需用户点确认**，避免误识别发布/上课时间 |
| 疑似 Key/Token | 常见前缀（`sk-`/`ghp_`/`AKIA` 等） | 默认折叠 + 点开 + 一键复制 |

**V1 不引入**：大模型、向量数据库、RAG。

---

## 7. 链接处理

保存 URL 后自动尝试抓取：页面标题、域名、favicon、描述、封面图。
尽量兼容：B站 / GitHub / 小黑盒 / 微信公众号。
**抓取失败仍保存原始 URL，绝不阻塞发送。**

**V1 不做**：正文存档、离线阅读、页面快照、自动摘要、自动内容分类。

---

## 8. 图片与文件

- 支持：图片 / PDF / DOCX / Markdown / TXT / 压缩包 / 配置文件 / 其他常见文件
- 功能：图片缩略图+预览、文件名+大小、下载、移动端系统分享、删除、保留 MIME 与扩展名
- 附件存 NAS 挂载目录，不入库
- **不内置打印**。作业打印流程 = 电脑上传 → 手机下载/分享 → 转发微信打印小程序

---

## 9. 统一时间线（首页）

类聊天信息流：

- 按创建时间倒序；支持今天/昨天/更早分组
- 新发送内容立即出现；无限滚动或分页；移动端桌面端都好用

每条记录支持：一键复制文字、下载/分享附件、编辑、删除、置顶、转待办、设/改 DDL、标完成、加简单分类、看原始链接。

---

## 10. 输入与发送

- 输入区：输入/粘贴文字、粘贴图片、拖拽文件、选择上传、粘贴 URL、移动端拍照/选图
- 发送：默认不弹表单；Enter 提交，Shift+Enter 换行；保留原始文件名+扩展名；**上传失败保留输入内容不丢失**

---

## 11. 搜索与筛选（核心能力）

- **搜索范围**：正文、链接标题、URL、域名、文件名、分类、创建时间、DDL
- **筛选**：内容类型、是否待办、是否完成、是否置顶、分类、DDL 范围
- V1 只要关键词搜索 + 按钮筛选，不要求复杂搜索语法（用 SQLite FTS5）

---

## 12. 登录与部署

- 部署：Docker Compose + NAS 本地存储 + 自有域名 + 反向代理 + HTTPS
- 用户：**单用户**，不开放注册，无邮件验证，无多人权限
- 认证（V1）：简单账号密码（session）；预留反代认证 / Authentik / Authelia / OIDC

---

## 13. 数据与备份

- SQLite + SQLite FTS5 全文搜索；NAS volume 存附件
- 支持：数据库定时备份、附件目录备份、JSON 导出、原始附件导出
- **V1 不需要**：PostgreSQL、Redis、消息队列、对象存储、多节点

---

## 14. 客户端策略

一套 Web 代码 + 各平台**原生薄壳**。壳只做：登录 + 展示 WebView + 接收系统分享 + 上传 + 再分享。**业务逻辑全在服务端。**

| 平台 | 壳方案 | 系统分享 | 里程碑 |
|---|---|---|---|
| 全平台兜底 | PWA / 浏览器 | Web Share Target | V1 |
| Android | Capacitor | `ACTION_SEND` | V1.1 |
| HarmonyOS NEXT | 原生 ArkTS | Share Kit | V1.1 |
| Windows / macOS | Tauri 2 | Share Extension / Share Target | V1.2 |
| iOS | Capacitor（按需） | Share Extension | 以后 |

**iOS 说明**：上架 App Store 需 $99/年 Apple Developer；单用户自用可 sideload（免费证书 7 天过期需重签，或付费转 1 年）。现在用不到，**不进 V1/V1.1，不为它花钱**，代码随时能补壳。

---

## 15. V1 MVP 必须实现（19 项）

1. 单用户登录　2. 发送文字　3. 粘贴+上传图片　4. 上传/下载/分享文件　5. 保存链接　6. 链接基础预览　7. 统一时间线　8. 一键复制　9. 编辑删除　10. 置顶　11. 标记待办　12. 设/改 DDL　13. 勾选完成　14. 全文搜索　15. 类型+状态筛选　16. 响应式移动端　17. PWA　18. Docker Compose 部署　19. SQLite + 附件备份

---

## 16. 后续增强（按价值排序）

1 手机系统分享 → 2 浏览器扩展/Bookmarklet → 3 DDL 自动识别 → 4 WebSocket 实时更新 → 5 桌面全局快捷输入 → 6 CLI → 7 自由标签 → 8 批量操作 → 9 导入微信历史 → 10 网页正文存档 → 11 AI 摘要/自动分类

---

## 17. 明确不做（V1）

多人协作 · 公开注册 · 商业化 · 独立密码保险箱 · 端到端加密 · 自动剪贴板监听 · AI Agent · RAG · 向量数据库 · 知识图谱 · 完整笔记编辑器 · Notion 式 Block · 完整日历 · 课程表 · 看板/甘特图 · 原生全平台大客户端 · 内置打印服务 · 社区/插件市场

---

## 18. V1 验收标准

1. Windows / macOS / HarmonyOS / Android 上均可正常访问
2. 从打开页面到发文字 ≤ 一次输入 + 一次点击
3. 文字/图片/文件/链接均能跨设备获取
4. 手机能把文件下载或分享给其他应用
5. 历史内容可关键词搜索找到
6. 任意记录可快速设待办 + 加 DDL
7. 即将截止内容可单独查看
8. API Key 等文本可快速复制
9. 所有内容仍在一条统一时间线中
10. 整体体验接近「微信发给自己」，但搜索与后续处理更强
